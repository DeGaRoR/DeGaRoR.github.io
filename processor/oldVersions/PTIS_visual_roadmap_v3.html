Phase 0
Rebase

Freeze authority and clean the specs

Before more content work, align the docs with the current engine and pick one canonical campaign structure.
spec + planning

Do

  • Update the mission system spec to current engine reality
  • Archive conflicting campaign fragments
  • Confirm template/profile/composite vocabulary

Output

  • single authoritative roadmap
  • single campaign ordering
  • clean spec baseline for implementation

Why now

  • prevents double work
  • avoids building against stale assumptions
  • makes later communication coherent
Medium effort · high leverage
Phase 1
Validate

Human-room validation harness

Treat the survivability model as a calibration module, not as a mission. Build the Ineluctable Fate harness first.
physics-critical

Core model

  • human metabolism rates
  • room as real thermodynamic volume
  • O₂ depletion / CO₂ rise / humidity / heat
  • death and incapacitation thresholds

Need immediately

  • time-series graphs
  • drag variables from inspector
  • preconfigured and user graphs
  • step / pause / export support

Success check

  • plausible survival timing
  • reasonable rate curves
  • variants: leaks, scrubber, O₂ injection, multiple humans
Highest priority · central to survival fantasy
Phase 2
Wrap

Composite units and readable abstraction

You need to hide internal complexity behind readable devices. This is where the game starts looking playable rather than merely inspectable.
usability-critical

First composites

  • habitat module
  • human occupant
  • greenhouse
  • other mission-facing assemblies

UI implications

  • show simplified icon and label
  • allow drilling into internals
  • support mission-specific profiles cleanly

Icon strategy

  • procedural ragtag SVG
  • no external image dependencies
  • profile-level icon parameters or inline SVG fragments
High priority · unlocks clarity and narrative packaging
Phase 3
Frame

Minimal mission kernel + UI shell

Do not manually build many missions before the shell exists. Build only what the first missions need: menu flow, mission loading, save/restore, objectives, and debrief.
product-critical

Mission kernel

  • briefing → build → run → evaluate → debrief
  • palette restrictions
  • starting state + objectives + fail conditions
  • mission as data file

UI shell

  • welcome screen
  • Missions / Sandbox entry points
  • return home, save, restore
  • title/logo placeholders that still look intentional

Rule

  • prefer data over scripting
  • small escape hatch only if needed
  • avoid bespoke mission plumbing
High priority · prevents building content twice
Phase 4
Slice

Planet X vertical slice: Missions 1–5

This is the first meaningful external object. It should prove tension, readability, and progression.
alpha candidate

Mission arc

  • survive water
  • survive oxygen
  • survive fuel/power
  • stabilize atmosphere pressure/quality
  • sustain sustained survival loop

Must be felt

  • room matters
  • humans matter
  • power deficits matter
  • flow visibility makes reasoning intuitive

Gate

  • non-engineer can finish M1 unsupervised
  • systems players see depth
  • the game looks like a product, not a dev harness
External milestone · first serious alpha target
Phase 5
Grow

Focused alpha testing + early audience building

Start communicating before broad marketing. Use the game both as a game pitch and as a demonstration of serious engineering and systems design capability.
external-facing

Key people

  • systems players: Factorio / ONI / Kerbal types
  • engineers and ex-colleagues
  • professors / educators
  • a few smart non-technical players

Communication

  • short clips every 2–3 weeks
  • Reddit for curiosity and raw reaction
  • LinkedIn for credibility and consultancy signaling
  • small, targeted outreach over broad promotion

How to gauge enthusiasm

  • show mission concepts as one-screen mockups
  • poll between scenarios
  • track comments, saves, DM interest, volunteer testers
  • measure which fantasies people repeat back to you
Starts once the slice is real enough to defend
Phase 6
Complete

Finish Planet X M6–M10, then expand by best interest/effort ratio

The backbone remains the 10-mission Planet X campaign. After that, pick the next pack by appeal and reuse, not by novelty alone.
campaign completion

Finish core

  • stabilize cold chain
  • stabilize power scaling
  • stabilize fertilizer
  • sustain cryo / biosphere closure

Then likely next

  • real space engineering pack
  • ISS / starship life support
  • Mars ISRU
  • Mars base expansion

Hold later

  • terraforming as a distinct later layer
  • Earth challenge pack after core proves itself
  • far-future infrastructure once packaging is strong
Only after the backbone genuinely works

Strategic tracks

What runs in parallel without diluting the main path

Technical

Keep investing where it directly increases tension, clarity, or mission reuse.

  • human-room validation
  • graphs and diagnostics
  • composites
  • mission data architecture

Presentation

Enough shell and visual language to avoid “prototype shame” during alpha.

  • welcome screen
  • intentional logo/title area
  • mission cards / scenario language
  • ragtag procedural icon system

Testing

Use three complementary groups, not one.

  • simulation players
  • engineers / professors / ex-colleagues
  • non-technical comprehension testers

Communication

Soft, repeated proof of progress beats big announcements.

  • short clips
  • small devlogs
  • scenario polling
  • LinkedIn + targeted communities

Decision rules

To keep scope sane

Ship discipline

  • Do not build broad mission packs before the mission kernel exists.
  • Do not let terraforming redefine the product before Planet X is fun.
  • Do not let consultancy signaling distort what makes the game playable.
  • Every major addition must improve either readability, tension, or campaign reuse.

What to watch closely

  • Whether the habitat loop feels visceral or still mostly numerical.
  • Whether Mission 1 can be understood without supervision.
  • Whether custom profile icons remain procedural, safe, and maintainable.
  • Whether outside interest clusters around specific scenario fantasies.
